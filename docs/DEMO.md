# 答辩演示指南

## 推荐流程（约 5 分钟）

1. 登录 `/admin`，在“活动设置”点击“重置演示数据”，展示 `100 → 80 → 50 → 35` 的实时漏斗。
2. 解释分散在问卷、邮箱和云平台的数据，如何通过统一任务入口沉淀为同一参与者的连续轨迹。
3. 打开“签到入口”，展示每个阶段自动生成的二维码和链接。
4. 用手机扫描“活动报名”，以一个新学号报名；刷新后台后展示报名人数变化。
5. 扫描“账号开通”完成下一阶段签到，再次提交同一信息，说明重复签到不会重复计数。
6. 打开“参与者”查看任务轨迹并导出 CSV，展示运营无需再跨表 VLOOKUP。
7. 在“活动设置”将活动切换为“已结束”，再次扫码展示入口关闭，同时后台历史数据仍可查询。

## 核心讲述

系统把原本滞后的人工聚合改为统一入口实时采集。运营人员不仅能看到最终提交数量，还能定位账号开通、首次任务等具体流失节点，并及时采取运营动作。

## 演示前检查

- 确认公网 URL、管理员账号和数据库健康。
- 用手机网络实际扫描一次二维码，避免仅在 ECS 内网可访问。
- 提前生成演示数据，检查漏斗显示 100、80、50、35。
- 准备一个未使用的学号和手机号用于现场报名。
- 不要在真实业务环境点击“重置演示数据”。
